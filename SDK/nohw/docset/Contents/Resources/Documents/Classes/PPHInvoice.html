NSString The itemId for the new invoice item. Must be unique to other invoice items which don’t have any itemDetailId. You can use this to get a pointer to the item later using invoiceWithId if you don’t call reconcileInContext. If the itemId is not unique this method will return nil. unitPrice

taxRate

taxRateName

NSString The name for the tax rate given to this item.

– addItemWithId:detailId:name:quantity:unitPrice:taxRate:taxRateName:

Creates an invoice item with the given parameters an returns a pointer to it. ONLY use this function if you want to make an item that is not bound to a SellableItem for some reason. If you add one of these such items to the invoice and later call reconcileInContext the item will be associated with a new SellableItem and the item’s itemId will be set to that of the SellableItem, rendering the previous itemId useless for searching in the invoice.

- (PPHInvoiceItem *)addItemWithId:(NSString *)itemId detailId:(NSString *)detailId name:(NSString *)name quantity:(NSDecimalNumber *)quantity unitPrice:(NSDecimalNumber *)unitPrice taxRate:(NSDecimalNumber *)taxRate taxRateName:(NSString *)taxRateName

Parameters

itemId

NSString The itemId for the new invoice item. Must be unique to other invoice items (including itemDetailId). You can use this to get a pointer to the item later using invoiceWithId if you don’t call reconcileInContext.
name

NSString The item name.
quantity

NSDecimalNumber The number of the items to put in the cart.
unitPrice

NSDecimalNumber The price of a single one of these items.
taxRate

NSDecimalNumber The tax to be applied to this item.
taxRateName

NSString The name for the tax rate given to this item.
itemDetailId

NSString The “identifying details” for the item such as size, toppings, etc.

Return Value

PPHInvoiceItem The newly created item.

Discussion

Creates an invoice item with the given parameters an returns a pointer to it. ONLY use this function if you want to make an item that is not bound to a SellableItem for some reason. If you add one of these such items to the invoice and later call reconcileInContext the item will be associated with a new SellableItem and the item’s itemId will be set to that of the SellableItem, rendering the previous itemId useless for searching in the invoice.
