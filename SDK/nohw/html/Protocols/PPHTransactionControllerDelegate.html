onPostAuthorize

- (void)onPostAuthorize:(BOOL)didFail

Parameters

didFail
  • indicates whether the authorization failed. A true indicates that the authorization itself failed. A false indicates that the authorization did not fail. It however, DOES NOT indicate that the transaction was successful.

Discussion

onPostAuthorize

This callback is invoked by the TransactionManager once authorization is complete. Note that authorization complete does not indicate success. The input parameter indicates whether or not the authorization failed.

onPreAuthorizeForInvoice:withPreAuthJSON:

onPreAuthorize This callback is invoked by the TransactionManager in response to a call to the processPayment() API in the TransactionManager. As soon as the TransactionManager is ready to perform the authorization this callback is invoked. The SDK is responsible for ensuring that this call is invoked in the appropriate thread.

- (PPHTransactionControlActionType)onPreAuthorizeForInvoice:(PPHInvoice *)inv withPreAuthJSON:(NSMutableDictionary *)preAuthJSON

Parameters

inv

This is the Invoice object against which the authorization will be performed. At this point the application can choose to make some changes to the invoice. For example, a tip could be added to an Invoice at this time or an item could be removed etc.

preAuthJSON

A dictionary ready for json conversion and shipment to the paypal service that represents the request payload. This is the request data that will be sent to the backend and the application is allowed to modify this data. CAUTION: Modifying the data incorrectly could cause your transaction to fail. Ensure that any changes by the app will not introduce errors and that you handle the error cases appropriately. More documentation on what the server expects in the payload can be found in the documentation.

Return Value

TransactionControlAction. Return HANDLED if you want the SDK to stop processing the transaction at this point If HANDLED is returned, then the Transaction will be canceled but the application is free to keep the invoice around if it so wishes. If the invoice is not required then use the cancel method defined on the Invoice to cancel it. Return CONTINUE if you wish the SDK to continue processing this transaction.

Discussion

onPreAuthorize This callback is invoked by the TransactionManager in response to a call to the processPayment() API in the TransactionManager. As soon as the TransactionManager is ready to perform the authorization this callback is invoked. The SDK is responsible for ensuring that this call is invoked in the appropriate thread.
